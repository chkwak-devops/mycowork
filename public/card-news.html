AI Design System Guide

DESIGN.md로
시작하는 AI 디자인
시스템 가이드

똑같은 AI UI는 이제 그만. DESIGN.md 한 장이면 당신의 브랜드가 살아납니다.

스크롤
1
⚠

AI에게 "예쁘게 만들어줘"는
통하지 않는 시대

AI 코딩 어시스턴트에게 UI를 맡기면 항상 비슷한 결과물이 나옵니다. 라운드 카드, 보라-파랑 그라데이션, 중앙 정렬 히어로섹션… 나쁘진 않지만, "우리 브랜드"는 아닙니다.

기존 방식
"버튼 하나 만들어줘"
→ 보라색 둥근 버튼
→ 모든 프로젝트가 비슷비슷
→ 브랜드 정체성 없음
DESIGN.md 방식
"DESIGN.md 보고 만들어줘"
→ Stripe 스타일의 버튼
→ 브랜드 컬러/타이포 일관
→ "아, 우리 제품 맞네"

문제는 AI가 당신의 디자인 맥락을 모른다는 겁니다. Figma 파일은太重하고, 설명은 번거롭습니다. 여기서 탄생한 개념이 바로 DESIGN.md입니다.

2
📐

DESIGN.md, 개념은
단순합니다

Google Stitch가 제안한 DESIGN.md는 디자인 시스템을 마크다운 한 장으로 정의하는 포맷입니다.

DESIGN.md = AI가 읽는 디자인 명세서

컬러 팔레트, 타이포그래피, 간격, 컴포넌트 스타일, 그림자, 분위기 — 모든 디자인 결정을 평문 마크다운으로 기록합니다. Figma 플러그인도, JSON 스키마도 필요 없습니다.

핵심은 이것입니다: AI 에이전트가 코드를 생성하기 전에 DESIGN.md를 읽으면, 브랜드에 일관된 UI를 만들어냅니다.

DESIGN.md가 담는 것

  • 디자인 철학 — "미니멀리즘", "모노크롬 프리시전" 등 브랜드 분위기
  • 컬러 토큰 — Primary, Secondary, Accent, Background, Text
  • 타이포그래피 — 폰트 패밀리, 사이즈 스케일, 자간, 행간
  • 간격 & 레이아웃 — spacing unit, border radius, container max-width
  • 컴포넌트 규칙 — 버튼, 카드, 입력창, 네비게이션 스타일
  • 분위기 — 섀도우, 블러, 그라데이션, 모션 방향

4
🚀

사용 방법은 딱 두 단계

  1. DESIGN.md 가져오기

    터미널에서 아래 명령어를 실행하거나, getdesign.md에서 직접 다운로드하세요.

    # Vercel 스타일 적용
    npx getdesign@latest add vercel

    Stripe라면 npx getdesign@latest add stripe, Linear라면 npx getdesign@latest add linear

  2. AI에게 DESIGN.md를 참고하라고 말하기

    선택한 DESIGN.md 파일이 프로젝트 루트에 추가됩니다. 이후 AI에게 이렇게 말하세요:

    "이 프로젝트의 DESIGN.md를 읽고, 그 스타일을 따라 landing page를 만들어줘."

5
🎨

내 브랜드의 DESIGN.md는
직접 만들 수 있습니다

awesome-design-md의 템플릿을 사용하는 것에서 한 걸음 더 나아가, 우리 브랜드만의 DESIGN.md를 만들 수도 있습니다.

방법 1: designmd.me 사용 (추천)

URL 하나만 넣으면 AI가 자동으로 DESIGN.md를 생성해줍니다.

designmd.me → 사이트 URL 입력 → DESIGN.md + HTML Preview 자동 생성

npx @crowdlinker/designmdme stripe.com 으로 CLI 사용도 가능

방법 2: 직접 작성

기본 구조는 간단합니다. 프로젝트 루트에 DESIGN.md 파일을 만들고, 컬러/타이포/컴포넌트 규칙을 마크다운으로 적으면 됩니다.

# Design System

## Colors
- Primary: #6366f1
- Background: #f8f7f4
- Text: #1a1a2e

## Typography
- Font: "Noto Sans KR"
- Heading weight: 700
- Body size: 15px

자세한 스펙은 Google Stitch DESIGN.md 명세를 참고하세요.

6
💡

실전 사용 꿀팁

DO DESIGN.md는 프롬프트의 시작점

"DESIGN.md를 보고 만들어줘" + "특히 이 부분은 이렇게" 구체적인 지시를 함께 주면 결과물이 더 좋습니다.

프롬프트 예시

"DESIGN.md를 참고해서 pricing 페이지를 만들어줘. 특히 카드 디자인은 그림자 대신 border 스타일을 쓰고, CTA 버튼은 초록색 accent 컬러로 강조해줘."

DO 여러 DESIGN.md를 조합

Vercel의 미니멀한 타이포그래피 + Stripe의 카드 디자인을 섞고 싶다면? DESIGN.md 두 개를 참고하라고 하면 AI가 알아서 조합합니다.

# 프로젝트에 여러 DESIGN.md 보관
project-root/
├── DESIGN.md # 주 디자인
├── DESIGN.vercel.md # 보조 참고
└── DESIGN.stripe.md # 컴포넌트 참고

DON'T DESIGN.md를 방치하지 마세요

프로젝트가 발전하면 디자인도 변합니다. DESIGN.md도 함께 업데이트해야 AI가 최신 스타일을 유지할 수 있습니다. 리팩토링할 때 DESIGN.md도 같이 리뷰하는 습관을 들이세요.

TIP org 단위로 공유하세요

팀 레포지토리의 .github/DESIGN.md에 공용 디자인 시스템을 정의하면, 모든 팀원이 동일한 브랜드 경험을 유지할 수 있습니다.

7
👥

팀에 적용하는
3단계 로드맵

  1. 1주차: 파일 하나 추가하기

    프로젝트 루트에 DESIGN.md 하나만 추가하세요. 팀 공유 없이 개인이 AI에 지시할 때만 사용해도 효과가 있습니다.

  2. 2주차: 팀 공유 & 피드백

    팀 채널에 DESIGN.md를 공유하고 "AI가 이 스타일을 따라 코드를 짭니다"라고 알리세요. 각자 수정 제안을 받아 개선합니다.

  3. 3주차: PR 템플릿에 포함

    "이 PR은 DESIGN.md를 준수했는가?"를 리뷰 체크리스트에 추가합니다. AI 생성 코드의 디자인 일관성을 유지하는 최종 관문입니다.

궁극의 목표

"Figma 없이 DESIGN.md만으로 AI 에이전트가 브랜드에 완벽하게 일관된 UI를 생성하는 것."

이제 더 이상 디자이너에게 "여백 좀 늘려주세요"라고 말할 필요가 없습니다. DESIGN.md에 "spacing: 8px 단위"라고 써 놓으면 AI가 알아서 합니다.

8
🔮

이렇게 달라집니다

# Before — DESIGN.md 없이
"landing page 만들어줘"
→ 보라색 히어로, 둥근 버튼, 센터 정렬, 무난한 카드
# After — DESIGN.md 적용
"DESIGN.md 보고 landing page 만들어줘"
→ Vercel 스타일: 흑백 프리시전, Geist 폰트, border 기반
→ Stripe 스타일: 브랜드 보라, 카드 그림자, 넉넉한 여백
→ Linear 스타일: 타이트한 자간, 미니멀, 모노크롬

AI의 기본값에 의존하지 않고, 당신의 브랜드가 AI에게 말을 거는 시대입니다. DESIGN.md 한 장이면 충분합니다.
